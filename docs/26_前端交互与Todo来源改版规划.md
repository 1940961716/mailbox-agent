# 前端交互与 Todo 来源改版规划

## 改版背景

原前端将邮件、Todo、草稿、附件、日志全部展开在首页，更适合开发调试，但不像真实邮箱产品。用户在真实邮箱场景中更习惯“左侧列表、右侧阅读”的交互模式。

## 改版目标

将系统改造成邮箱式工作台：

- 左侧：邮箱同步、测试提示、邮件列表。
- 右侧：选中邮件的内容和 Agent 处理结果。
- 右侧顶部：展示邮件主题、发件人、时间、分类、优先级。
- 右侧 Tab：原文、Agent 结果、全部 Todo、回复草稿、附件、日志。
- 左侧增加“处理全部邮件”，用于批量运行 Agent。
- 增加独立 `Todo 总览` 页面，而不是把全局 Todo 混在邮件详情里。
- Todo 总览支持点击统计卡片进行状态筛选，并保留优先级筛选。
- Todo 默认按优先级排序：高 -> 中 -> 低；同优先级按截止时间排序。
- 邮箱同步改成类似登录界面。
- IMAP 服务器改为下拉选择：QQ、163、126、Gmail、Outlook、自定义。
- 支持本机记住账号；授权码可由用户选择是否记住。

## Todo 来源设计

Todo 不是孤立任务，必须明确来源：

- 来源邮件主题。
- 来源发件人。
- 来源邮件 ID。
- 原文证据 evidence。
- Critic Agent 审查意见。

Todo 是所有邮件处理后的综合结果，不只属于当前选中的邮件。前端 Todo 展示格式：

```text
任务标题
来源：邮件主题 / 发件人
证据：邮件原文或附件行
截止：...
优先级：...
审查：...
```

## 交互验收标准

- 用户同步邮箱后，可以在左侧列表选择邮件。
- 右侧默认展示邮件原文。
- 点击“运行 Agent”后，右侧 Agent 结果区显示 Triage、Action、Critic 三步。
- Todo 区域展示所有邮件生成的综合 Todo，并明确每条任务的来源。
- 当前选中邮件对应的 Todo 在全局 Todo 中高亮。
- 日志区域只展示当前邮件对应的 Agent 日志。
- 用户可以进入 Todo 总览，点击统计卡片切换全部、待确认、已确认、已完成，并按优先级筛选任务。
- 用户可以在邮箱授权区选择邮箱类型，不必手动输入常见 IMAP 服务器。
- 用户可以选择是否在本机记住账号和授权码，并能清除记忆。

## 自动化边界

Agent 自动完成：

- 邮件分类。
- Todo 生成。
- 回复草稿生成。
- 附件分析。
- Critic 审查。

Agent 不自动完成：

- 最终发送邮件。
- 删除邮件。
- 修改真实邮箱状态。

最终发送邮件属于高风险动作，应由用户确认后再执行。
