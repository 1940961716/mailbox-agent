# 邮件发送功能评估

## 当前是否可以发送邮件

当前系统本体不提供“发送回复邮件”功能。

已有能力：

- `scripts/send_test_emails.py` 可以通过 163 SMTP 发送测试邮件。
- 前端可以生成回复草稿。
- 前端可以保存和复制回复草稿。

未实现能力：

- 在系统中点击按钮直接发送 Agent 生成的回复草稿。

## 是否应该做发送功能

MVP 阶段不建议自动发送邮件。

原因：

- 邮件发送是不可逆动作，风险高。
- LLM 可能误解语气、对象或上下文。
- 课程项目更需要展示 Agent 推理、Todo 生成和人机协同，而不是追求自动发信。

## 推荐设计

如果后续要做，建议采用“显式确认发送”：

1. Agent 只生成草稿。
2. 用户手动编辑草稿。
3. 用户点击“发送”。
4. 系统弹出二次确认。
5. SMTP 发送。
6. 记录发送日志。

## 建议优先级

| 功能 | 优先级 | 说明 |
|---|---|---|
| 回复草稿生成 | P0 | 已实现 |
| 草稿保存 / 复制 | P0 | 已实现 |
| SMTP 发送测试邮件 | P1 | 已实现脚本 |
| 前端发送草稿 | P2 | 后续可做 |
| 自动发送邮件 | 不做 | 不符合人机协同安全边界 |
