# 真实邮箱 Agent 功能测试指南

## 测试前准备

1. 使用专门的测试邮箱，不建议使用个人主邮箱。
2. 在邮箱设置中开启 IMAP/SMTP。
3. 获取邮箱授权码，不要使用登录密码。
4. 启动系统并访问：

```text
http://127.0.0.1:8001
```

## 页面测试流程

1. 在左侧填写邮箱账号、授权码，选择邮箱类型。
2. 选择同步时间范围，例如“最近 7 天”或“自定义范围”，并设置最多读取封数作为安全上限。
3. 点击“按时间同步”。
3. 在邮件列表中选择一封邮件。
4. 点击“运行 Agent”。
5. 观察“邮件详情与 Agent 结果”中的三步：
   - Triage 分类：判断邮件类型、优先级、是否要处理。
   - Action 执行：生成 Todo、回复草稿、附件分析结果。
   - Critic 审查：检查风险、模糊时间、低置信度字段。
6. 查看 Todo、回复草稿、附件分析和 Agent 日志。

## 建议发送到测试邮箱的邮件

### 测试 1：任务邮件

主题：

```text
【任务】请在本周五前提交项目演示视频
```

正文：

```text
你好，请在本周五下班前提交智能邮箱 Agent 的项目演示视频。
视频需要包含邮件同步、Agent 分类、Todo 生成、草稿生成和附件分析。
完成后请回复邮件确认。
```

预期结果：

- 分类为任务类。
- 生成 Todo。
- 截止时间被解析或提示确认。
- 生成回复草稿。
- Critic 检查时间和任务完整性。

### 测试 2：需要回复的邮件

主题：

```text
请确认 MVP 是否支持附件分析
```

正文：

```text
你好，请确认当前版本是否支持读取 Excel 或 CSV 附件，并把表格中的任务转成 Todo。
如果可以，请回复一个简短说明。
```

预期结果：

- 分类为需要回复或任务类。
- 生成回复草稿。
- Todo 可选生成。
- Agent 日志中能看到 Action Agent 输出。

### 测试 3：会议邮件

主题：

```text
周四下午项目评审会议通知
```

正文：

```text
本周四下午 3 点进行项目评审会议，请准备 5 分钟演示。
重点说明敏捷过程文件、Agent 工作流和当前可运行功能。
如无法参加请提前回复确认。
```

预期结果：

- 分类为会议类。
- 生成准备演示的 Todo。
- 可生成回复草稿。

### 测试 4：附件任务表邮件

主题：

```text
本周项目任务表，请根据附件推进
```

正文：

```text
团队好，附件中是本周项目任务表，请根据 owner 和 deadline 推进。
高优先级任务请优先完成，明天下班前同步一次进展。
```

附件建议使用 CSV 或 Excel，字段如下：

```csv
task,owner,deadline,priority
完成 IMAP 邮件同步,后端开发,2026-06-12,high
完善 Agent 日志展示,前端开发,2026-06-13,medium
准备答辩演示脚本,项目负责人,2026-06-14,medium
```

预期结果：

- 邮件被识别为任务类。
- 附件分析区域显示字段和摘要。
- 表格每一行可生成 Todo。
- Critic 审查低置信度或模糊字段。

### 测试 5：普通通知邮件

主题：

```text
课程资料更新通知
```

正文：

```text
课程资料已更新到平台，请有需要时自行查看。本邮件无需回复。
```

预期结果：

- 分类为通知类。
- 不应生成高优先级 Todo。
- 不应生成强制回复草稿。

## 怎么判断 Agent 确实工作了

看四个地方：

1. 邮件详情中的“Agent 做了什么”。
2. Todo 区域是否生成结构化任务。
3. 回复草稿区域是否生成可编辑草稿。
4. Agent 日志中是否出现：
   - Triage Agent
   - Action Agent
   - Critic Agent

如果日志里的 Action Agent 输出包含 `LLM Action Agent 输出`，说明当前使用的是 DeepSeek LLM。

如果日志里显示 `规则型 Action Agent 输出`，说明 LLM 调用失败或服务没有加载 `.env`，需要检查 Key 并重启服务。
