# 29 全量审查整改记录

日期：2026-06-10

## 背景

根据“智能邮箱助理与自动化待办生成 Agent”的原始开题规划书、项目落地 TodoList 和当前产品实现，进行了一次全量审查。审查发现当前 MVP 已具备真实邮箱同步、Agent 分类、Todo 生成、回复草稿、附件分析、全局 Todo 总览和测试邮件脚本等核心能力，但在 Todo 生命周期、用户编辑闭环、LLM 透明日志和批量处理控制方面仍需补强。

## 本轮整改目标

1. 重跑 Agent 时不再删除用户已经确认、完成或拒绝的 Todo。
2. Todo 总览页补齐编辑和删除能力。
3. LLM 调用失败时必须在 Action Agent 日志中体现降级原因。
4. 邮箱授权码记忆改为明确的本机演示模式，并提示 localStorage 明文保存风险。
5. 批量处理增加范围和数量控制，避免默认扫过过多真实邮件。

## 已完成修改

### 1. Todo 持久状态保护

- 新增 `clear_generated_email_outputs`，只清理 `pending` Todo 和未编辑的 `draft` 草稿。
- 保留 `confirmed`、`done`、`rejected` 状态的 Todo。
- 新增 `Persistence Guard` 日志，记录本次重跑清理了多少临时结果、保留了哪些用户决策状态。
- 新增持久 Todo 重复保护：当同一封邮件中已经存在确认/完成/拒绝的同名任务时，重跑 Agent 不再重复创建。

### 2. Todo 编辑与删除

- 后端新增 `DELETE /api/todos/{todo_id}`。
- 前端 Todo 卡片新增“编辑”和“删除”按钮。
- 编辑弹窗支持修改标题、说明、截止日期、优先级、状态和证据。
- 状态筛选仍通过 Todo 页统计卡片完成，符合当前交互要求。

### 3. LLM 失败可见化

- `llm_service` 记录最后一次 LLM 调用失败原因。
- `Action Agent` 输出中增加 `llm_status`、`llm_error`、`llm_model`、`llm_provider` 字段。
- 当 DeepSeek/OpenAI-compatible 调用失败或未配置 Key 时，日志明确显示已使用规则兜底。

### 4. 邮箱授权记忆安全提示

- 默认推荐只记住邮箱账号和服务器。
- 保存授权码改名为“本机演示模式保存授权码”。
- 保存授权码前弹出确认，说明授权码会明文保存到当前浏览器 `localStorage`。
- 清除记忆时同时清除账号、授权码和授权码保存确认状态。

### 5. 批量处理控制

- 批量运行 Agent 支持选择：
  - 只处理未处理邮件。
  - 重新处理全部邮件。
- 支持设置最多处理数量。
- 支持演示场景下对每封邮件强制生成 Todo。
