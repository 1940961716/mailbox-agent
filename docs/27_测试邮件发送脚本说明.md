# 测试邮件发送脚本说明

## 目标

使用 163 邮箱作为发件邮箱，向 QQ 测试邮箱发送一组专门设计的测试邮件，用于验证智能邮箱 Agent 的分类、Todo 生成、回复草稿、附件分析和 Critic 审查能力。

## 配置方式

在项目根目录创建或修改 `.env`：

```env
SMTP_HOST=smtp.163.com
SMTP_PORT=465
SMTP_SSL=true
SMTP_USER=你的163邮箱@163.com
SMTP_AUTH_CODE=你的163邮箱授权码
TEST_TO_EMAIL=你的QQ测试邮箱@qq.com
```

注意：

- `SMTP_AUTH_CODE` 不是邮箱登录密码。
- 需要在 163 邮箱设置里开启 SMTP/IMAP 服务并生成授权码。
- 建议使用专门的测试 163 邮箱，不要使用个人主邮箱。

## 查看有哪些测试邮件

```powershell
.\.venv\Scripts\python.exe scripts\send_test_emails.py --list
```

## Dry-run 预览

默认不会真实发送，只会打印将要发送的邮件：

```powershell
.\.venv\Scripts\python.exe scripts\send_test_emails.py
```

## 真实发送全部测试邮件

```powershell
.\.venv\Scripts\python.exe scripts\send_test_emails.py --send
```

## 只发送某几封

```powershell
.\.venv\Scripts\python.exe scripts\send_test_emails.py --case task,reply,attachment --send
```

## 测试邮件类型

| key | 场景 | 预期 Agent 行为 |
|---|---|---|
| task | 明确任务邮件 | 生成 Todo 和草稿 |
| reply | 需要回复邮件 | 生成回复草稿 |
| meeting | 会议通知 | 生成准备会议 Todo |
| attachment | 带 CSV 附件任务表 | 分析附件并生成多个 Todo |
| ambiguous_time | 模糊时间 | Critic 提醒人工确认时间 |
| security | 安全提醒 | 生成安全跟进 Todo |
| notification | 普通通知 | 可不生成 Todo，或由用户强制生成 |

## 发送后如何测试

1. 打开系统：`http://127.0.0.1:8001`。
2. 使用 QQ 测试邮箱同步邮件。
3. 选择带 `[Agent测试]` 前缀的邮件。
4. 点击“运行 Agent”。
5. 查看：
   - Agent 结果
   - Todo 来源
   - 回复草稿
   - 附件分析
   - 日志
