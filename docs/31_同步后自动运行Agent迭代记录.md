# 31 同步后自动运行 Agent 迭代记录

日期：2026-06-11

## 需求来源

用户反馈：邮箱同步之后没有自动生成 Todo 和回复草稿，需要额外点击“批量处理”或单封“运行 Agent”，这与项目目标中的“智能邮箱助理自动完成邮件分类、Todo 生成、草稿生成和附件处理”不一致。

## 设计决策

同步邮件不应只是“拉取邮件列表”，而应触发完整 Agent 工作流：

1. IMAP 按时间范围同步邮件。
2. 刷新本地邮件快照。
3. 自动批量运行 Agent。
4. 自动生成 Todo、回复草稿和附件分析。
5. 自动跳转到 Todo 总览，展示本次同步后的综合结果。

仍然保留安全边界：系统只生成回复草稿，不自动发送邮件。

## 已完成修改

- 前端 `syncMailbox()` 在同步成功且 `imported > 0` 后，自动调用 `POST /api/emails/process-all`。
- 自动处理范围为本次同步后仍处于 `new/unprocessed` 状态的邮件。
- 自动处理完成后刷新邮件、Todo、草稿、附件和日志。
- 自动切换到 Todo 总览页。
- 同步按钮文案从“按时间同步”改为“同步并运行 Agent”。
- 若本时间范围内没有邮件，则不会运行 Agent，并提示用户改选最近 7 天或最近 30 天。
- 前端静态资源版本号更新，避免浏览器缓存旧脚本。

## 验收点

1. 用户点击“同步并运行 Agent”。
2. 如果同步到邮件，系统自动执行 Triage、Action、Critic、附件分析和保存流程。
3. Todo 总览自动出现本次邮件生成的任务。
4. 回复草稿在对应邮件详情的“回复草稿”页签中可查看和编辑。
5. 不需要再手动点击“批量处理”才能看到 Todo。

## 注意

不是每一封邮件都会生成 Todo 或草稿。通知、安全提醒、广告类邮件可能只生成分类和日志。只有 Agent 判断需要行动或需要回复时，才会生成 Todo 或草稿。
